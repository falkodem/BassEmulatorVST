# BassEmulatorVST — Roadmap & Research Notes

## Цель проекта

Трансформировать гитарный звук в звук баса в реальном времени.
Основная проблема Guitar Rig / Ampero Stomp: неточное интонирование при атаке ноты.
Сценарий использования: монофонная игра, запись бас-партий с помощью гитары.

---

## Текущее состояние

- [x] MVP: JUCE plugin wrapper + реверб (VST3, работает в Reaper)
- [x] Phase 1: Классический bass emulator (YIN + onset + envelope + sawtooth + LadderFilter)
- [ ] Phase 2: RTNeural — TCN для real-time (требует парные данные гитара/бас)
- [ ] Phase 3: RAVE — VAE для offline постобработки

## Research: Рыночные решения

| Решение | Подход | Проблема |
|---|---|---|
| Boss OC-2 (железо) | Аналоговый делитель частоты | Быстро, но синтетический звук |
| EHX POG (железо) | Цифровой pitch shifter | Задержка трекинга на атаке |
| Ampero Stomp (железо) | DSP + pitch detection | Та же задержка на атаке |
| Guitar Rig (софт) | Pitch detection + ресинтез | Та же проблема |

Корень проблемы: все классические решения используют явный шаг pitch detection,
который требует несколько периодов сигнала — а на атаке сигнал ещё не периодический.

## Research: Классические алгоритмы pitch detection

| Алгоритм | Точность | Латентность | Выбран? |
|---|---|---|---|
| Zero-crossing | Плохая | ~0 мс | Нет |
| Autocorrelation | Средняя | 20–40 мс | Нет |
| **YIN** | **Хорошая** | **~21 мс (окно 1024)** | **Да** |
| pYIN | Отличная | 30–50 мс | Нет (сложнее) |
| FFT-based | Средняя | 50–100 мс | Нет |

**YIN выбран** как оптимальный баланс точности и сложности реализации для монофонного сигнала.

### Важно: детекция на гитарном сигнале, не на басовом

Pitch detector работает на входном гитарном сигнале (E2–E4), а не на синтезируемом басе (E1–E3).
Это означает более короткие периоды и меньшую требуемую латентность:

| Нота гитары | Частота | Период | Окно для YIN |
|---|---|---|---|
| E2 (нижняя струна) | 82 Гц | 12 мс | ~1024 сэмплов (21 мс @ 48kHz) |
| A2 | 110 Гц | 9 мс | ~512 сэмплов (10 мс) |
| D3 | 147 Гц | 7 мс | ~512 сэмплов (10 мс) |

**Вывод:**
- Для записи (с компенсацией латентности в DAW): окно 1024 сэмплов (~21 мс) — покрывает все ноты гитары
- Для мониторинга real-time: окно 512 сэмплов (~10 мс) — уверенно от A2 вверх, E2 с небольшой погрешностью

## Research: RAVE (Realtime Audio Variational autoEncoder)

**Авторы:** Antoine Caillon, Philippe Esling (IRCAM, 2021)
**GitHub:** https://github.com/acids-ircam/RAVE

Архитектура: VAE (Variational AutoEncoder)
- Encoder сжимает аудио в латентный вектор (тембр, стиль, динамика)
- Decoder разворачивает латентный вектор обратно в аудио
- Ключевая инновация: каузальная архитектура → стриминговый режим

Применение для проекта:
- Обучить на записях баса → модель учится "пространству звуков баса"
- На инференсе: encoder извлекает характеристики из гитары, decoder генерирует бас
- Тембральная трансформация + атака решаются имплицитно через обучение

**Статус: отложено на Phase 3 (offline режим)**
Причины: требует GPU для обучения, большой датасет, сложнее интеграция с JUCE.
Кандидат для тяжёлого offline режима постобработки.

## Research: End-to-end нейронная сеть

Идея: обучить модель напрямую waveform гитары → waveform баса.
Модель сама учится обрабатывать атаку — нет явного pitch detection шага.

**Кандидат архитектуры: TCN (Temporal Convolutional Network)**
- Хорошо работает с аудио (используется в amp-sim плагинах)
- Поддерживается RTNeural
- Фиксированная предсказуемая задержка (в отличие от RNN/LSTM)
- Эффективнее LSTM для длинных зависимостей

**Статус: Phase 2**
Нужно: обучающие данные (парные записи гитара/бас), эксперименты с архитектурой.

## Phase 1: Классический bass emulator — дизайн

### Пайплайн

```
Вход (гитара)
     │
     ├──────────────────────────────┐
     │                              │
     ▼                              ▼
[Onset detector]            [YIN pitch detector]
 "новая нота?"               F0 = 147 Hz (D3)
     │                              │
     │                    F0 / 2 = 73.5 Hz (D2)
     │                              │
     │                              ▼
     │                    [Sawtooth oscillator]
     │                    генерирует D2 + все гармоники
     │                              │
     └──────────────────────────────┤
                                    ▼
                          [Envelope follower]
                          огибающая с гитары → динамика баса
                                    │
                                    ▼
                          [Resonant lowpass filter]
                          срезает верха → тембр баса
                                    │
                                    ▼
                               Выход (бас)
```

### Параметры плагина (Phase 1)

| Параметр | Диапазон | Назначение |
|---|---|---|
| Filter Cutoff | 100–2000 Гц | Яркость/тёмность тембра |
| Filter Resonance | 0–1 | Характер атаки фильтра |
| Envelope Attack | 1–50 мс | Скорость нарастания |
| Envelope Release | 10–500 мс | Скорость затухания |
| Dry/Wet | 0–1 | Смешение гитары и баса |

### Обработка атаки (Phase 1)

Стратегия: предсказание по предыдущей ноте.
При степпинге (ноты идут последовательно без перекрытий) — держим последний
стабильный F0 пока YIN не выдаст новый стабильный результат.
Атака для MVP не является приоритетом.

### JUCE компоненты

- `juce::dsp::Oscillator` — sawtooth генератор
- `juce::dsp::LadderFilter` — резонансный lowpass (Moog-стиль), уже в juce_dsp
- Envelope follower — написан сам (простая RC-цепь)
- YIN — написан сам

### DSP компоненты — детали реализации

**YinPitchDetector** — кольцевой буфер 1536 сэмплов (1024 + 512), обновление питча каждые 1024 новых сэмпла. 4 шага: difference function → CMNDF → absolute threshold (0.15) → parabolic interpolation. Диапазон: 70–1300 Гц (гитарный регистр).

**OnsetDetector** — RMS текущего блока vs предыдущего. Порог: ratio > 2.0 (6 дБ), пол тишины: 1e-4. Cooldown 50 мс после срабатывания.

**EnvelopeFollower** — one-pole filter с раздельными attack/release коэффициентами. `coeff = exp(-1 / (sampleRate × ms / 1000))`. `triggerAttack()` сбрасывает envelope в 0 для чёткой атаки.

**Oscillator** — `juce::dsp::Oscillator`, sawtooth: `f(x) = x / π`. Частота = F0/2 (октава вниз).

**Filter** — `juce::dsp::LadderFilter`, режим LPF12.

## Phase 2: Plan — RTNeural

### 1. Архитектура решения

**Решено:**
- Формат: WAV, **44.1 кГц** (совпадает с sample rate плагина, избегаем ресэмплинга), mono
- Вид данных: **waveform** — старт с простого, spectral domain если не хватит качества
- Три разных понятия размера:
  - **Inference chunk** (processBlock) = 512–1024 сэмплов
  - **Receptive field** модели = 8192–32768 сэмплов (реализуется через dilated conv, не размером входа)
  - **Training sequence** = 16000–65000 сэмплов
- Нижняя нота входа — E2 (82.41 Гц = ~582 сэмпла при 48k); 1024-сэмпловый чанк покрывает ~1.7 периода E2 — достаточно для инференса

**Открыто:**
- Синхронизация пар гитара/бас — скрипт выравнивания, решается в п.2

### 2. Датасет

**Источники баса (в порядке предпочтения для iteration 1):**
1. **Bass VST + MIDI** — чистый тембр, идеальная атака (Trilian, Session Bassist, LABS Bass)
2. Сплайс: MIDI атака (~30 мс) + crossfade на реальный хвост октавера
3. Октавер с разметкой `has_good_attack` — использовать только хорошие семплы

Реальный бас добавить позже — улучшит тембр, но не структуру.

**Костяк (iteration 1) — только для stateless CNN (RF = 21 мс):**
- Single notes по всему диапазону E2–E4
- Хроматические гаммы
- Несколько типовых бас-партий (2–3 бара)
- ~10–15 минут синхронных пар

Легато, пиццикато, слэп — позже.

**Синхронизация:** оба инструмента на один клик в одном DAW-проекте. Критично — исправить постфактум невозможно.

**Структура хранения:**
```
dataset/
  guitar/   ← все гитарные файлы
  bass/     ← все басовые файлы
  index.csv
```

**Колонки index.csv:**
```
filename_guitar, filename_bass,
type,              # single_note, gamma, solo, ...
source,            # real, synth, external
freq_cluster,      # low (<150Hz), mid (150-400Hz), high (>400Hz)
bpm_cluster,       # slow (<80), mid (80-140), fast (>140)
tempo_bpm,         # точное значение
note_range_min_hz,
note_range_max_hz,
duration_sec,
guitar_slug,       # на будущее (разные гитары)
pickup_type,       # single, humbucker
pickup_active,     # true/false
tuning,            # standard, drop_d, ...
string_gauge,      # 9-42, 10-46, ...
interface,         # модель аудиоинтерфейса
gain_db,
has_good_attack,   # критично для октавер-записей
quality_rating     # 1–5, субъективно
```

**Аугментации:**
- Pitch shift ±2 ст. (оба файла синхронно — сохраняет F0→F0/2)
- Gain variation ±6 дБ
- Light clipping (имитация жёсткой игры)
- Noise floor variation (особенно для single coil)
- Time stretch ±2% (вариация темпа)

Реверб, дисторшн — нет (работаем с сырым сигналом).

### 3. Архитектуры для испытания

Фундаментальная проблема: октава вниз — это генерация когерентной суб-гармоники f₀/2,
которой нет во входном сигнале. Линейный фильтр её создать не может. Каждый подход
решает эту проблему по-своему.

| # | Название | Суть | Статус |
|---|---|---|---|
| v0 | Stateless DilatedCNN | Короткие окна 23 мс, учится через multi_scale_stft | ✅ В работе |
| v1 | Длинные окна | То же, но window_size=4096 (~93 мс ≈ 4 периода баса) | Следующий |
| v2 | Тембральная коррекция | YIN→synth_bass как input, модель учит только тембр | Приоритет |
| v3 | TCN (RTNeural) | Рекуррентное состояние между чанками, когерентная фаза | Phase 2 |
| v4 | RAVE / DDSP | VAE/дифференцируемый DSP, offline постобработка | Phase 3 |

---

**v1 — Длинные окна (4096 сэмплов)**
Оставить тот же stateless подход, поднять `window_size` до 4096.
~93 мс = ~4 полных периода баса E1 (41 Гц) — достаточно для когерентной суб-гармоники
внутри одного окна. Нужно: перерезать датасет, растянуть dilations до 2048,
пересмотреть `fft_sizes` в MultiScaleSTFTLoss.
Трейдофф: латентность ~93 мс (приемлемо для записи в DAW с компенсацией).

**v2 — Тембральная коррекция синтезированного баса**
Разделить задачи. YIN из Phase 1 детектирует f₀ гитары → синтезируется чистый бас
на f₀/2 (осциллятор, как сейчас в VST). ML-модель берёт пару `(guitar, synth_bass)`
и учит **только тембральную коррекцию**: убрать механический звук осциллятора,
добавить реалистичный атак, резонанс корпуса.

```
guitar ──→ YIN → f₀/2 → synth_bass ─┐
                                      ├──→ [ML] ──→ realistic_bass
guitar ───────────────────────────────┘
```

Когерентность гарантирует осциллятор (детерминированно). Модель не генерирует
суб-гармонику — только корректирует тембр. Датасет: тройки `(guitar, synth_bass, real_bass)`,
`synth_bass` генерируется оффлайн через YIN + осциллятор из VST-кода.

**v3 — TCN с состоянием (RTNeural)**
Модель хранит hidden state между processBlock вызовами → фаза суб-гармоники
когерентна между чанками. RTNeural поддерживает GRU/LSTM в C++.
Трейдофф: сложнее BPTT-обучение, нужна осторожная инициализация состояния.

---

- **Два режима:**
  - Real-time: лёгкая модель, жёсткий бюджет латентности (< 10 мс) — v2, v3
  - Offline: тяжёлая модель, качество важнее скорости — v4
- **Loss:** Multi-scale STFT (текущий), добавить perceptual/hybrid для v2+
- **Метрики:** как оценить "звучит как бас" объективно — TBD

### Sample Rate

Датасет записан и обрабатывается на **44100 Гц**. Плагин (Phase 1) использует `getSampleRate()` из DAW и работает на любом SR. Модель Phase 2 обучена на 44100 Гц — для инференса Reaper должен быть настроен на 44100 Гц.
